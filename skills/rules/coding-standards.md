# 代码规范

## 命名
| 类型 | 规范 |
|------|------|
| 变量/函数 | camelCase |
| 类/组件 | PascalCase |
| 常量 | UPPER_SNAKE_CASE |
| 文件 | PascalCase |

## 目录结构原则
- 功能域划分：按功能域划分，不按文件类型平铺
- 模块自包含：业务模块自包含，不跨模块直接引用内部文件
- 公共提升：可复用内容提升到公共层
- 结构约定：具体目录结构在 `dev-planning` 阶段约定

## 强制规则
- 常量定义：常量单独定义，禁止魔法数字和魔法字符串
- 复用提取：重复逻辑/样式提取为公共函数/组件/样式变量，不允许复制粘贴
- 参数限制：参数超过 3 个改用对象传参
- 提交清洁：不提交 console.log、堆积的 TODO
- 错误处理：所有错误必须有明确处理，禁止静默失败（空 catch、吞掉异常）
- 注释原则：注释说明"为什么"，不说"做什么"
- 函数/类注释：所有函数和类必须有注释，说明用途、参数和返回值
- 关键逻辑：复杂或非直觉的逻辑必须用注释说明执行过程
- 第三方优先：遇到问题优先使用成熟且近期有更新的第三方库，而不是自己实现
- 敏感信息：禁止在代码中硬编码密码、密钥、token 等敏感信息，统一通过环境变量管理
