# UI 设计工具规范

## 推荐工具

- [Pencil](https://www.pencil.dev/) — 当前推荐，支持 MCP 集成

## 步骤

按步骤一步一步操作，每一步都先输出标题，再执行

- 选择工具：选择设计工具，默认使用 Pencil
- 环境检查：确认工具已安装
- 设计：逐模块进行，每个模块确认后再继续下一个
  - 创建设计文件：如果对应位置没有对应的设计文件，按[目录结构]要求，使用[所选工具]创建设计文件
  - 打开设计文件：用[所选工具]打开刚创建的文件
  - 绘制设计稿：在已打开的设计文件中，使用[所选工具]完成该模块设计；每次修改后，都需要保存文件，避免丢失
  - 保存设计文件
- 导出素材：设计完成后，用[所选工具]将以下内容分别导出为独立切片，存放到 `assets/` 目录
  - 导出范围：整页、每个独立模块/区块、可复用组件（按钮、输入框组、图标等）
  - 文件命名：`<模块名>-<描述>.png`，使用小写字母和连字符，禁止使用节点 ID 作为文件名
    - 示例：`register-page.png`、`register-submit-btn.png`、`register-field-email.png`
  - 导出完成后列出所有文件路径确认

## Pencil 使用说明

- 创建文件：用文件系统工具创建 `.pen` 文件，内容写入 `{}`
- 打开文件：用 Pencil MCP 的 `open_document`
- 绘制设计稿：通过 Pencil MCP 完成；内容变更后，需要保存.pen文件

## 目录结构

设计文件必须严格按以下结构存放：

```
designs/
  <模块名>/
    <模块名>.<ext>  ← 设计文件
    assets/      ← 素材目录
```
