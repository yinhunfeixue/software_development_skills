# 任务拆解规范

## 步骤

> 每个步骤先输出标题，再输出结果

- 读取设计文档：读取软件设计阶段的输出文档，获取模块划分和模块依赖关系
- 拆解：将每个模块拆分为小功能，直到每个任务可独立开发和 review
- 粒度检查：检查每个任务是否能在一次开发迭代内完成，过大的继续拆分
- 依赖关系：基于模块依赖关系，用表格标注任务间的依赖，无依赖的任务不需要列出，示例：

  | 任务 | 依赖 |
  |------|------|
  | 登录 | 注册 |
  | 下单 | 登录, 支付接口 |

- 输出任务清单：列出所有任务和依赖关系表，等待开发者确认

## 注意事项

- 独立性：每个任务可独立开发、review 和合并
- 无循环依赖：任务间不能有循环依赖
